# LinkedIn Post Draft

Built **PulsePath Stadium OS** for **PromptWars Virtual** under the **Physical Event Experience** challenge.

The idea: what if a stadium could guide every attendee like a personal copilot while giving the venue team a real-time command center for crowd flow, wait times, accessibility reroutes, and emergency coordination?

So I built a full-stack web app with:

- an attendee assistant that recommends the best live route for entry, food, restrooms, merch, accessible seating, or fast exits
- an ops dashboard that can switch live scenarios and generate announcement-ready reroute guidance
- deterministic routing logic first, with Gemini layered on top for natural explanations
- Firebase/App Hosting-ready deployment and Firestore-ready realtime sync

The demo includes 4 scenarios:

- Pre-entry Rush
- Halftime Surge
- Accessible Reroute
- Emergency Exit Coordination

What I loved most about this build was using AI for intent-driven development while still being ruthless about product clarity, fallback safety, and user experience.

This was a fun one to design because I wanted it to feel less like a generic dashboard and more like a sports-broadcast control room.
